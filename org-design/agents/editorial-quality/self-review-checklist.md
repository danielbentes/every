# Self-Review Checklist for Editorial Quality Agent

Before presenting any article assessment, verify against these gates:

### Article Publication Gate (Blocking)
Full criteria: read `gates/article-publication.md`

- [ ] All five Tier 1 criteria evaluated with specific findings (not just PASS/FAIL labels)
- [ ] All three Tier 2 criteria evaluated with advisory notes
- [ ] Every AI tell detected includes a line reference and suggested alternative phrasing
- [ ] Thesis located in first 3 paragraphs or actual location flagged
- [ ] At least one concrete experience example identified or absence documented
- [ ] Assessment text does not contain AI tells ("Moreover," "Furthermore," "It's worth noting")
- [ ] Assessment does not use banned Every phrases ("leverage," "synergy," "content" for articles, "users" for readers)
- [ ] Builder credibility claims checked against actual Every experience
- [ ] Overall recommendation (PASS/REVISE/ESCALATE) consistent with individual criteria results
- [ ] Notification formatted correctly and posted to #editorial within 5 minutes
- [ ] Decision logged to decision ledger with agent ID, tier, action, and rationale

## Notes
- Gates derive from: `gates/article-publication.md` + role-specific editorial responsibilities
- Always consult the full gate file for complete criteria before running assessment
- Never read `gates/.holdouts/article-publication-holdouts.md` — holdout scenarios exist to test this agent
