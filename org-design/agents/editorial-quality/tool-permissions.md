# Tool Permissions for Editorial Quality Agent

| Action | Permission | Condition |
|--------|-----------|-----------|
| Article text analysis | Autonomous | Read-only access to drafts in editorial pipeline |
| AI tells detection patterns | Autonomous | Katie Parrott's encoded criteria library |
| Rigor test evaluation | Autonomous | Kate Lee's three rigor tests as structured checks |
| Every article archive search | Autonomous | For originality comparison against published articles |
| Slack #editorial notifications | Autonomous + Notify | Post assessment summaries within 5 min of completion |
| Decision ledger append | Autonomous | Log all Tier 2+ decisions per governance spec |
| Author voice profiles | Autonomous | Read-only reference for voice consistency |
| Linear editorial board | Autonomous | Read-only: check article status, deadlines, revision history |
| Publication systems (CMS, newsletter, social) | Never | Publish is always human-operated |
| Author direct messages | Never | All communication through Kate or Eleanor |
| Editorial calendar / strategy docs | Never | Editorial direction is Tier 4 (Kate only) |
| Consulting or product data | Never | Out of scope for editorial role |

## Notes
- Authority matrix mapping: primarily Tier 1-2 with Tier 3 for publication recommendations
- Filtered for editorial domain scope
- Maps to `governance/AUTHORITY-MATRIX.md` editorial agent entries
