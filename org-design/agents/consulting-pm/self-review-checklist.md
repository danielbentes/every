# Self-Review Checklist for Consulting PM Agent (Claudie)

Before requesting Natalia's approval on any client-facing deliverable, verify against these gates:

### Consulting-Deliverable Gate (Blocking)
Full criteria: read `gates/consulting-deliverable.md`

- [ ] All Tier 1 gate criteria pass (grounded experience, no unfamiliar tools, no overselling, client confidentiality)
- [ ] All Tier 2 gate criteria pass (client customization, actionable recommendations, hands-on component)
- [ ] Client confidentiality verified: zero references to other engagements, other clients, or cross-engagement patterns
- [ ] Every recommendation references specific Every experience (named team member or named product/tool)
- [ ] At least one recommendation is implementable by the client in their first week without Every's help
- [ ] At least one hands-on building exercise is included or referenced
- [ ] No language implies financial commitments, discounts, or contract terms
- [ ] Materials are customized to this client's industry, team size, and AI maturity level
- [ ] The deliverable review request format is complete with all gate status fields
- [ ] NPS-relevant signals are tracked and surfaced
- [ ] The decision was logged to the decision ledger

### Status Report Gate (Blocking)

- [ ] Report covers only the engagement it's scoped to -- no cross-engagement data
- [ ] Blockers and risks are flagged prominently
- [ ] Report is formatted for Natalia's 5-minute review workflow
- [ ] Notification sent within 5 minutes of generation

## Notes
- Gates derive from: `gates/consulting-deliverable.md` + role-specific consulting responsibilities
- Always consult the full gate file for complete criteria before running assessment
- Never deliver to client without Natalia's explicit approval, regardless of gate results
